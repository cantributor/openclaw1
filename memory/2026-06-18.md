# 2026-06-18

## Бадди — Keenetic Buddy 6 (mesh-ретранслятор Wi-Fi 6 AX3000)

Продолжили начатую вчера (17 июня, вечер) настройку. «Бадди» = этот девайс.

### Топология
- Основной роутер — **Huawei**, `192.168.3.1`, проводной интернет, раздаёт Wi-Fi. Ноут Can (`192.168.3.12`) подключён к нему по Wi-Fi (`wlp61s0`), подсеть `192.168.3.0/24`.
- **Buddy 6** работает в режиме **extender**: uplink-станция (`WifiStation0`) цепляется к Wi-Fi-сети **«Садовая 69»** (НЕ к Huawei напрямую) и сам раздаёт две свои сети.
- Keenetic — не mesh-хозяин (его нет), поэтому Buddy как обычный extender зашёл в сеть клиентом.

### Доступ к Buddy
- IP **плавает по DHCP**: 17 июня был `192.168.3.22`, 18 июня — `192.168.3.23`. Искать по MAC.
- **MAC Buddy: `56:ff:20:e3:a5:64`** (Keenetic-префикс семейства `5x:ff:20`).
- Веб-морда не пускает: порт 80 → HTTP 403, `my.keenetic.net` → 502, 443 не проверяли толком. Управление — через **Telnet (порт 23)**.
- Консоль: KeeneticOS 4.01, hostname `Keenetic-a564`, логин `admin` + пароль администратора (Can знает, мне не говорит). Приглашение `(config)>`.
- Выход из telnet: `exit` (или `Ctrl+]` → `quit`).

### Как нашли IP (повторяемый рецепт)
Без установки nmap/arp-scan, ARP-скан вручную:
```
ip route | grep default; ip -4 addr show | grep "inet "   # убедиться, что в сети 192.168.3.0/24
for i in $(seq 1 254); do ping -c1 -W1 192.168.3.$i >/dev/null 2>&1 & done; wait; echo "=== соседи ==="; ip neigh | grep 192.168.3 | sort -t. -k4 -n
```
Buddy опознаём по MAC `56:ff:20:e3:a5:64`.

### Диагноз проблемы
- Сети, которые раздаёт Buddy: `concord-buddy-2.4` (2.4 ГГц, `AccessPoint`) и `concord-buddy-5` (5 ГГц, `AccessPoint_5G`).
- На обеих было `encryption wpa2`, но **строки `authentication wpa-psk` (ключа) НЕ было** → сеть «защищённая», но пароль не подходит никакой. Битое состояние после мастера настройки (поля пароля были залочены с пометкой «такой же», но ключ не записался).
- Уточнение Can: пароль в мастере был недоступен для ввода, помечен «такой же» (как у усиливаемой сети). По факту наследование не сработало.

### Что сделали (исправление)
В telnet `(config)>`:
```
interface AccessPoint authentication wpa-psk <ПАРОЛЬ_от_Садовая_69>
interface AccessPoint_5G authentication wpa-psk <ПАРОЛЬ_от_Садовая_69>
system configuration save
```
- Пароль Can вводил сам (открытым текстом в команде), мне не показывал.
- `show running-config interface <name>` НЕ работает на этой прошивке (`argument parse error`) — смотреть только полный `show running-config`.
- После записи у `concord-buddy-2.4`, `concord-buddy-5` и uplink `WifiStation0` («Садовая 69») хеш совпал: `authentication wpa-psk ns3 uDtwqph9wH57rtmAiB0TeZrt`. Значит пароль на всех трёх одинаковый = пароль «Садовой 69».
- `ns3 ...` — это **зашифрованный** (обфусцированный) вид пароля Keenetic, НЕ открытый. В команды как пароль подставлять нельзя.
- Конфиг сохранён (`Core::System::StartupConfig: Saving (cli).`).
