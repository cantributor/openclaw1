## Починка памяти (memory_search)

Обнаружено: я не помнил вчерашнюю работу. Две независимые причины:
1. Перестал вести дневные заметки — последняя правка памяти была 29 мая.
2. `memory_search` был сломан: дефолтный провайдер эмбеддингов — OpenAI, ключа нет → семантический поиск отключён.

Важное про эмбеддинги: Anthropic (Claude) **не даёт API эмбеддингов вообще**. Claude — разговорная модель, эмбеддер — отдельный класс. Поэтому Claude не закрывает задачу поиска по памяти.

Решение (вариант «локально, бесплатно», в духе whisper):
- Установлен `node-llama-cpp` 3.18.1 в `~/.npm-global/lib/node_modules/openclaw/node_modules` (`npm install node-llama-cpp --no-save`). Пакет openclaw вырос 369М → 1.4Г (prebuilt-бинари).
- В `~/.openclaw/openclaw.json` → `agents.defaults.memorySearch.provider = "local"`. Бэкап: `openclaw.json.bak.*`.
- Модель эмбеддингов: `embeddinggemma-300m` (GGUF q8, ~0.3 ГБ), тянется с HuggingFace при первом запуске.
- Перезапущен gateway. Проверено: `memory_search` возвращает результат с vectorScore — работает локально, без внешних ключей. Первый запрос ~38 сек (прогрев модели), дальше быстрее.

Урок: писать итог каждой значимой сессии в `memory/ГГГГ-ММ-ДД.md`. Поиск теперь это найдёт.
